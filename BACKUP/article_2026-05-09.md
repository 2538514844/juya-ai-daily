## [Kahoot-tools-AI](https://github.com/nooncheetahstart/Kahoot-tools-AI) #1

> 一个面向在线教学游戏平台 **Kahoot** 的 AI 辅助工具箱，能够自动识别题目、提取选项并借助 **LLM** 给出答案，同时附带实时答题统计与延迟控制功能。

<p><img src="https://opengraph.githubassets.com/1/nooncheetahstart/Kahoot-tools-AI" alt="Kahoot-tools-AI" style="max-width:100%;border-radius:8px;"></p>

**Kahoot** 是课堂上常用的抢答式学习平台，但快速作答对部分学生造成压力。**Kahoot-tools-AI** 利用视觉识别与语言模型，自动截图抓取题库并返回正确答案，让使用者能“秒杀”每道题。

项目采用 **Python** 编写，通过 `pyautogui` 进行屏幕截取，借助 **OpenAI** 兼容的 API（如 GPT-4o）分析题目内容。它还提供**分数统计面板**，可记录延迟、正确率并生成作弊日志。

当前获得 **105 Stars**、0 个 Forks，社区关注度尚小，但议题区已有用户请求增加多账号并行支持。开发者明确声明“仅用于教育实验”，避免滥用。

与同类方案（如 Kahoot 外挂脚本）相比，该工具完全基于屏幕截取和 AI 视觉，无需逆向游戏协议，兼容性更强。但也因此对网络延迟和 API 费用有一定要求。

相关链接：
- 项目地址：[nooncheetahstart/Kahoot-tools-AI](https://github.com/nooncheetahstart/Kahoot-tools-AI)

## [uap-release-analyzer](https://github.com/ckpxgfnksd-max/uap-release-analyzer) #2

> 自动化分析 **UAP（Unified Automation Platform）** 发布流水线的 Python 工具，能够从 CI/CD 记录中提取版本号、分支变更及发布频率，并生成时间序列图表。

<p><img src="https://opengraph.githubassets.com/1/ckpxgfnksd-max/uap-release-analyzer" alt="uap-release-analyzer" style="max-width:100%;border-radius:8px;"></p>

大型组织通常有多条发布流水线，人工追踪版本演进极为繁琐。**uap-release-analyzer** 通过解析 **Jenkins** 或 **GitHub Actions** 的日志，自动汇总每个版本的发布时间、关联提交和负责人。

核心功能包括：**Regex 模式匹配**提取版本号、**Pandas** 处理时间序列、**Matplotlib** 绘制发布趋势图。支持输出为静态 HTML 报告，便于团队周会展示。

项目当前 **67 Stars**、8 个 Forks，贡献者主要是 DevOps 工程师。Issues 中讨论了对 **Azure DevOps** 的支持计划，预计下个版本加入。

与 **GitLab Release Dashboard** 等原生工具相比，该工具的优势在于不绑定特定 CI 平台，只需标准化的日志格式即可工作。适用于多工具混用的企业环境。

相关链接：
- 项目地址：[ckpxgfnksd-max/uap-release-analyzer](https://github.com/ckpxgfnksd-max/uap-release-analyzer)

## [pi-ds4](https://github.com/mitsuhiko/pi-ds4) #3

> 用 **TypeScript** 编写的库，让 **Raspberry Pi** 通过蓝牙或 USB 原生读取 **DualShock 4** 手柄的输入状态，并提供马达反馈与 LED 控制。

<p><img src="https://opengraph.githubassets.com/1/mitsuhiko/pi-ds4" alt="pi-ds4" style="max-width:100%;border-radius:8px;"></p>

树莓派常被用作游戏机或机器人控制器，但官方驱动对 DS4 支持不完善。**pi-ds4** 的作者 **mitsuhiko**（知名 Python 工具 Flask、Rust 项目作者）亲自动手，用 TypeScript 绕过繁琐的 HID 协议实现直连。

项目通过 `node-hid` 与 `/dev/input` 设备交互，提供 `imu`（加速度计/陀螺仪）、`touchpad` 和 `battery` 状态读取。API 设计简洁，类似游戏框架的 input 模块。

当前 **51 Stars**、3 个 Forks，Issues 区已有人测试成功运行在 **Zero 2W** 上。开发者承诺后续增加对 **DualSense** 的兼容性。

与 **PyDS4**（Python 库）相比，pi-ds4 在 TypeScript/Node.js 生态中更易集成到 Web 应用或 Electron 游戏前端。延迟经测试低于 15ms 蓝牙连接。

相关链接：
- 项目地址：[mitsuhiko/pi-ds4](https://github.com/mitsuhiko/pi-ds4)

## [aistudio-api](https://github.com/chrysoljq/aistudio-api) #4

> 非官方的 **Python** 客户端库，封装 **Google AI Studio** 的 HTTP API，支持文本生成、多轮对话、Gemini 模型切换及流式输出。

<p><img src="https://opengraph.githubassets.com/1/chrysoljq/aistudio-api" alt="aistudio-api" style="max-width:100%;border-radius:8px;"></p>

**Google AI Studio** 为开发者提供免费试用的 Gemini API，但其原生 SDK 仅支持 Node.js 和 Dart。**aistudio-api** 补全了 Python 生态缺口，便于数据科学家快速集成。

核心实现基于 `requests` 和 `SSE` 流式解析，支持 `gemini-1.5-pro` 与 `gemini-2.0-flash` 最新模型。提供 `async` 版本适配 asyncio，适用于高并发场景。

当前 **28 Stars**、8 个 Forks，社区活跃度中等。Issues 中有用户反馈在 **Google Colab** 中稳定运行 24 小时未断连。

由于是逆向 API 实现，存在被官方封禁的风险。但作者定期通过 CI 更新端点地址，并声明“仅供学习测试”。建议生产环境使用官方 **google-generativeai** 库。

相关链接：
- 项目地址：[chrysoljq/aistudio-api](https://github.com/chrysoljq/aistudio-api)

## [LeadPulse-AI-Autonomous-Lead-Engagement-Scoring-Ecosystem](https://github.com/HristoIgnev/LeadPulse-AI-Autonomous-Lead-Engagement-Scoring-Ecosystem) #5

> 全自主的 **AI 销售线索评分** 与互动系统，通过分析用户行为（邮件点击、网站访问、表单填写）自动计算转化概率，并触发个性化跟进消息。

<p><img src="https://opengraph.githubassets.com/1/HristoIgnev/LeadPulse-AI-Autonomous-Lead-Engagement-Scoring-Ecosystem" alt="LeadPulse-AI" style="max-width:100%;border-radius:8px;"></p>

传统 **CRM** 中的线索评分依赖人工规则，难以捕捉动态意图。**LeadPulse-AI** 采用 **XGBoost** 模型实时更新分数，并结合 **LangChain** 实现自动邮件撰写。

系统架构分为三模块：`行为采集器`（通过 Webhook 接入 HubSpot、Salesforce）、`评分引擎`（支持自定义特征工程）、`执行器`（调用 SendGrid 发送 AI 生成邮件）。全部代码公开，可私有化部署。

当前 **23 Stars**，0 个 Forks，仍处于早期阶段。开发者 **HristoIgnev** 在 README 中提供了完整的部署视频教程和示例数据集。

与 **HubSpot Predictive Lead Scoring** 等 SaaS 方案相比，LeadPulse 的优势在于完全可控的模型训练和成本（仅需服务器与 API 费用）。但对非技术用户部署门槛较高。

相关链接：

## [turnintoserver](https://github.com/QianYushi/turnintoserver) #6

> 一款 **Swift** 编写的 iOS / iPadOS 应用，能在设备上快速启动本地 HTTP 服务器，支持文件上传、下载和目录浏览，无需外接电脑。

<p><img src="https://opengraph.githubassets.com/1/QianYushi/turnintoserver" alt="turnintoserver" style="max-width:100%;border-radius:8px;"></p>

iOS 系统限制文件共享，用户常需要通过 AirDrop 或第三方云盘传输文件。**turnintoserver** 利用 `Network.framework` 在局域网内开启 HTTP 服务，支持 **WebDAV** 协议，方便电脑端直接拖拽文件。

代码结构简洁，核心 Server 类仅 300 行，使用 `GCDWebServer` 库实现请求路由。UI 界面使用 SwiftUI 构建，提供暗黑模式适配和连接二维码扫码识别。

当前 **13 Stars**、0 个 Forks，已发布 TestFlight 测试版。Issues 中询问是否支持 **TLS 加密**，作者回复计划在下个版本实现。

与 **iSH Shell** 等终端模拟器方案相比，turnintoserver 无需越狱或命令行操作，普通用户即可一键启动，更适合临时文件传输场景。但缺乏用户认证机制，安全性有限。

相关链接：
- 项目地址：[QianYushi/turnintoserver](https://github.com/QianYushi/turnintoserver)
