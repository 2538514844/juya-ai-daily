## [Hermes Desktop OS](https://github.com/nickvasilescu/hermes-desktop-os1) #1

> **Hermes Desktop OS** 是一个用 **Swift** 编写的轻量级桌面操作系统核心，专注于极简架构和模块化设计。它尝试在低功耗硬件上提供流畅的桌面体验，目前处于早期开发阶段。

<p><img src="https://opengraph.githubassets.com/1/nickvasilescu/hermes-desktop-os1" alt="Hermes Desktop OS" style="max-width:100%;border-radius:8px;"></p>

该项目诞生于对现有桌面系统过于臃肿的不满。开发者希望用更现代的 **Swift** 语言重写操作系统核心，降低对系统资源的依赖。目前仅实现了基本的窗口管理、进程调度和输入处理。

技术方案上，**Hermes Desktop OS** 采用微内核架构，将驱动、文件系统、网络栈作为用户态服务运行。这种设计提高了系统的稳定性与安全性，但也带来了进程间通信的性能挑战。

截至 5 月 10 日，项目获得 **226 个 Stars** 和 **39 个 Forks**，在 GitHub 上引起了 Swift 和操作系统爱好者社区的关注。开发者计划每月发布一次 Alpha 版本。

与 Linux 微内核项目（如 **Minix**）不同，**Hermes Desktop OS** 在用户界面层直接使用 **SwiftUI** 编写，预期能更快构建原生视觉效果。适合对操作系统设计感兴趣的开发者学习或贡献。

- 项目地址：[nickvasilescu/hermes-desktop-os1](https://github.com/nickvasilescu/hermes-desktop-os1)

## [Sora-2 App No Watermark](https://github.com/baydarerhan/Sora-2-App-No-Watermark) #2

> **Sora-2 App No Watermark** 是一个基于 **HTML** 的简易封装应用，让用户直接调用 **OpenAI Sora** 生成视频，并去除默认水印。无需安装本地客户端，打开网页即可使用。

<p><img src="https://opengraph.githubassets.com/1/baydarerhan/Sora-2-App-No-Watermark" alt="Sora-2 App No Watermark" style="max-width:100%;border-radius:8px;"></p>

OpenAI 的 **Sora** 视频生成模型发布后迅速火爆，但官方生成的视频均带有水印。这个项目提供了一个替代前端，通过代理方式调用 Sora API，并在输出前移除水印。开发者称其仅用于学习和研究目的。

项目本质是一个单页 HTML 应用，包含简单的提示词输入框与生成结果展示区域。用户需要自备 Sora API Key。技术栈仅依赖浏览器，无需后端服务。

该项目获得 **115 个 Stars**，没有 Fork。社区讨论集中在水印移除的合规性上。开发者在 README 中特别声明不鼓励滥用。

与同类工具（如 **Sora Web UI**）相比，这个项目最大的特点是去水印功能，但稳定性取决于官方 API 的变动。适合希望在个人项目中避免水印的开发者试用。

- 项目地址：[baydarerhan/Sora-2-App-No-Watermark](https://github.com/baydarerhan/Sora-2-App-No-Watermark)

## [Mercury Agent Skills](https://github.com/cosmicstack-labs/mercury-agent-skills) #3

> **Mercury Agent Skills** 是一个面向 **Mercury Agent 框架** 的技能插件集合，允许开发者为 AI 智能体快速添加网页浏览、代码执行、文件操作等新能力。采用模块化设计，即插即用。

<p><img src="https://opengraph.githubassets.com/1/cosmicstack-labs/mercury-agent-skills" alt="Mercury Agent Skills" style="max-width:100%;border-radius:8px;"></p>

随着多智能体系统的兴起，开发者经常需要为 Agent 编写重复的功能代码。**Mercury Agent Skills** 旨在解决这个问题：它提供一组标准化的技能接口，只需简单配置就能让 Agent 拥有新技能。项目与 **Mercury Agent** 框架深度绑定。

技能库目前包含 **浏览器控制**、**Python 解释器**、**文件系统操作**、**HTTP 请求** 等基础技能。每个技能以独立包的形式发布，遵循统一的生命周期规范。

项目收获 **53 个 Stars** 和 **7 个 Forks**，社区贡献活跃。开发者计划后续加入数据库查询、邮件发送等企业级技能。

与 **LangChain** 的 Tool 插件相比，**Mercury Agent Skills** 更强调与 Mercury 框架的紧密集成，配置方式更简洁。适合已采用 **Mercury Agent** 的团队直接使用。

- 项目地址：[cosmicstack-labs/mercury-agent-skills](https://github.com/cosmicstack-labs/mercury-agent-skills)

## [minWM](https://github.com/shengshu-ai/minWM) #4

> **minWM** 是一个极简窗口管理器实现，核心代码仅数百行，支持基础的多窗口平铺与堆叠。专为嵌入式设备和轻量桌面环境设计，内存占用低于 10 MB。

<p><img src="https://opengraph.githubassets.com/1/shengshu-ai/minWM" alt="minWM" style="max-width:100%;border-radius:8px;"></p>

在桌面系统日趋复杂的今天，部分用户渴望回到纯粹、高效的窗口管理方式。**minWM** 由 **Shengshu AI** 团队开发，最初用于内部测试环境的轻量桌面，现在开源。

技术核心非常简单：直接操作 **X11** 或 **Wayland** 协议，实现窗口的创建、移动、缩放与关闭。没有动画、没有特效，但响应速度极快。启动时间小于 100 ms。

项目获得 **30 个 Stars**，尚未有 Fork。社区反馈中有人提到性能在树莓派上表现优秀。开发者正在考虑增加对 **dmenu** 和 **rofi** 的集成。

与 **DWM**、**i3** 等经典窗口管理器相比，**minWM** 的代码量更小，适合用于教学或嵌入式场景。不过功能也大幅缩水，不适合日常主力使用。

- 项目地址：[shengshu-ai/minWM](https://github.com/shengshu-ai/minWM)

## [Codex-app with DeepSeek](https://github.com/CAZAMA1/Codex-app-with-deepseek-perfectly) #5

> **Codex-app with DeepSeek** 是用 **Rust** 编写的 VS Code 扩展，将 **DeepSeek** 代码补全模型集成到 IDE 中。支持流式补全、多行建议，延迟低于 200 ms。

<p><img src="https://opengraph.githubassets.com/1/CAZAMA1/Codex-app-with-deepseek-perfectly" alt="Codex-app with DeepSeek" style="max-width:100%;border-radius:8px;"></p>

**DeepSeek** 作为国产开源大模型，在代码生成任务上表现接近 **GPT-4**。但官方提供的 VS Code 插件体验一般。这个项目重新实现了客户端，使用 **Rust** 编写核心逻辑以提高性能。

扩展的核心功能包括：触发补全时调用 DeepSeek API，在本地进行上下文缓存，支持自定义 prompt 模板。通过 **Rust** 的异步 IO 实现非阻塞补全。

该仓库获得 **26 个 Stars** 和 **3 个 Forks**。开发者声称补全延迟比官方插件快 3 倍，但未给出具体基准数据。社区有人指出需要 DeepSeek API Key。

与 **Continue.dev** 等通用代码 AI 扩展相比，**Codex-app with DeepSeek** 专为 DeepSeek 优化，配置更简单，但对其他模型不兼容。适合深度使用 DeepSeek 的开发者。

- 项目地址：[CAZAMA1/Codex-app-with-deepseek-perfectly](https://github.com/CAZAMA1/Codex-app-with-deepseek-perfectly)

## [GitMap v16](https://github.com/Poyeski/gitmap-v16) #6

> **GitMap v16** 是一个可视化 Git 仓库历史的工具，能以拓扑图形式展示分支结构、提交记录和贡献热度。支持交互式缩放与节点筛选。

<p><img src="https://opengraph.githubassets.com/1/Poyeski/gitmap-v16" alt="GitMap v16" style="max-width:100%;border-radius:8px;"></p>

开发者日常工作离不开 Git，但命令行和 GUI 工具（如 **Sourcetree**）在大型仓库中可视化效果有限。**GitMap** 采用不同的思路：将整个提交历史渲染为一张可缩放的地图，分支像河流交汇。

v16 版本新增了贡献热力环，用颜色深浅表示每位作者在某区域的活跃度。此外，支持导出为 SVG 和 PNG 格式。工具基于 **Electron** 构建，跨平台可用。

项目获得 **23 个 Stars**，开发者正在 GitHub 上收集功能建议。据介绍，加载拥有 10000 以上 commit 的仓库依然流畅。

与 **gitg**、**gitk** 等传统工具相比，**GitMap** 更注重视觉叙事，适合做技术分享或代码回顾时的演示工具。但实际代码审查场景中，功能还不够丰富。

- 项目地址：[Poyeski/gitmap-v16](https://github.com/Poyeski/gitmap-v16)

## [Python 速查指南](https://github.com/yayuvinagrawal-cloud/python) #7

> **Python 速查指南** 是一个面向编程初学者的 Python 语言速查表，涵盖基础语法、数据结构、函数、文件操作等核心知识点。采用 Markdown 编写，示例直观。

<p><img src="https://opengraph.githubassets.com/1/yayuvinagrawal-cloud/python" alt="Python 速查指南" style="max-width:100%;border-radius:8px;"></p>

很多 Python 新手在忘记语法时需要反复搜索，传统教程又太长。这个项目将最常用的 Python 知识点浓缩成一份文档，每个知识点附带一个即用示例。

内容涵盖：变量与类型、列表推导式、字典操作、异常处理、`lambda` 函数、模块导入等。文档以文件列表形式组织，可通过目录快速跳转。

该项目获得 **23 个 Stars**，全部来自个人项目展示。虽然星数不高，但内容质量得到一些评论认可。适合放在浏览器书签中随时查阅。

与 **Python cheat sheet**（如 **Real Python** 的版本）相比，此项目更精简，只包含最核心的部分。缺点是缺少 Python 3.11+ 的新特性，如 `match` 语句。

- 项目地址：[yayuvinagrawal-cloud/python](https://github.com/yayuvinagrawal-cloud/python)
