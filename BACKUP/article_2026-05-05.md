## [Fallout-Additions-Minecraft-Mod](https://github.com/pkmnono/Fallout-Additions-Minecraft-Mod) #1

> 《辐射》系列标志性的动力装甲、高斯步枪和辐射蝎即将降临 Minecraft 世界。该模组以忠实还原后末日美学为核心，为方块世界注入废土生存体验。

<p><img src="https://opengraph.githubassets.com/1/pkmnono/Fallout-Additions-Minecraft-Mod" alt="Fallout-Additions-Minecraft-Mod" style="max-width:100%;border-radius:8px;"></p>

模组作者 **pkmnono** 是《辐射》系列与 Minecraft 的双料爱好者，希望打破两个游戏的次元壁。项目自 2026 年初启动，目前处于早期开发阶段，已获得 84 个 Stars 和 0 个 Forks。

技术层面，模组利用 Minecraft Forge 框架，新增了 **T-60 动力装甲**、**胖子核弹**、**辐射蝎** 等 30 余种道具与生物。每种装备都拥有独特的属性和合成配方，例如动力装甲需要收集废铁与电路板打造。

社区反馈积极，玩家在 Issues 中要求增加更多《辐射 4》聚落建造元素。项目计划下一步加入 **辐射计量系统**，让角色在废土区域持续受到辐射伤害。

## [Fashion-AI](https://github.com/liangdabiao/Fashion-AI) #2

> 基于深度学习与风格迁移的智能穿搭推荐系统，仅需上传一张单品照片，即可生成数套完整搭配方案。项目由 **liangdabiao** 开发，采用 PyTorch 实现。

<p><img src="https://opengraph.githubassets.com/1/liangdabiao/Fashion-AI" alt="Fashion-AI" style="max-width:100%;border-radius:8px;"></p>

传统电商的搭配推荐多基于规则或协同过滤，难以理解颜色、材质与版型的视觉协调性。**Fashion-AI** 通过训练 ResNet-50 对时尚单品进行特征提取，再结合图神经网络构建搭配知识图谱。

核心功能包括“单品搭配生成”与“全身造型评分”。用户上传一件上衣，系统会推荐下装、鞋子和配饰，并给出整体评分。模型在 **Fashion-MNIST** 和自定义数据集上训练，Top-3 搭配准确率约 78%。

项目目前已收获 69 个 Stars，24 个 Forks，主要用 **Python** 编写。开发者提供了 Jupyter Notebook 演示和预训练权重，方便二次开发。未来计划接入电商 API 实现实时推荐。

相关链接：
- 项目地址：[liangdabiao/Fashion-AI](https://github.com/liangdabiao/Fashion-AI)

## [solidity-cot-auditor](https://github.com/butthtio/solidity-cot-auditor) #3

> 将 Chain-of-Thought（思维链）提示技术应用于智能合约安全审计，自动检测重入攻击、整数溢出等常见漏洞。由 **butthtio** 基于 GPT-4 API 构建。

<p><img src="https://opengraph.githubassets.com/1/butthtio/solidity-cot-auditor" alt="solidity-cot-auditor" style="max-width:100%;border-radius:8px;"></p>

传统静态分析工具如 Slither 存在误报率高、无法理解业务逻辑的痛点。**solidity-cot-auditor** 通过将合约代码拆分为多个步骤，引导大模型逐步推理合约执行路径，显著提升漏洞检出率。

工具接收 Solidity 源代码后，先进行语法解析，再生成 Chain-of-Thought 提示模板，最终输出漏洞类型、风险等级与修复建议。在公开的 SWC 漏洞测试集上，检测准确率达到 89%，召回率 76%。

该项目获得 69 个 Stars 和 54 个 Forks，社区参与度较高。开发者提供了命令行接口和 VS Code 插件，支持批量审计。代码完全开源，基于 **Python** 和 LangChain 框架。

相关链接：
- 项目地址：[butthtio/solidity-cot-auditor](https://github.com/butthtio/solidity-cot-auditor)

## [zyrln](https://github.com/ajavadinezhad/zyrln) #4

> 极简 Go 语言 HTTP 路由库，零外部依赖，性能对标 `fasthttp`。作者 **ajavadinezhad** 致力于提供最简洁的路由注册与中间件链。

<p><img src="https://opengraph.githubassets.com/1/ajavadinezhad/zyrln" alt="zyrln" style="max-width:100%;border-radius:8px;"></p>

Go 生态中路由库众多（`gin`、`chi`、`httprouter`），但部分库依赖较多且 API 复杂。**zyrln** 仅用不到 500 行代码实现了参数化路由、分组路由和中间件支持，核心逻辑基于前缀树（Trie 树）。

基准测试显示，在 1000 条路由场景下，`zyrln` 的单请求延迟约为 0.3μs，与 `http.ServeMux` 相比提升 40%。内存分配次数极低，适合对 GC 敏感的高并发场景。

项目目前有 41 个 Stars 和 8 个 Forks，采用 **Go** 语言编写。文档简洁，示例覆盖 RESTful API 和 WebSocket 路由配置，适合微服务框架的基础组件。

相关链接：
- 项目地址：[ajavadinezhad/zyrln](https://github.com/ajavadinezhad/zyrln)

## [MeetingBro](https://github.com/armpro24-blip/MeetingBro) #5

> 自动转录并总结视频会议内容的智能助手，支持 Zoom、Teams 和 Google Meet。使用 **Python** 编写，结合 Whisper 转录与 GPT 摘要。

<p><img src="https://opengraph.githubassets.com/1/armpro24-blip/MeetingBro" alt="MeetingBro" style="max-width:100%;border-radius:8px;"></p>

远程会议中手动记录笔记效率低且容易遗漏。**MeetingBro** 通过监听系统音频流（需虚拟音频设备）实时转录，并在会议结束后生成结构化摘要，包括讨论话题、决议和待办事项。

技术栈为 **OpenAI Whisper**（本地运行）用于语音转文字，**GPT-3.5-turbo** 用于文本摘要。支持中文和英文，可配置关键词高亮。项目还提供了浏览器插件用于自动获取会议链接。

目前获得 12 个 Stars 和 1 个 Fork，处于早期阶段。主要局限在于依赖本地 GPU 进行实时转录，开发者计划未来接入云端 API 以降低硬件要求。

相关链接：
- 项目地址：[armpro24-blip/MeetingBro](https://github.com/armpro24-blip/MeetingBro)

## [gptnb-image](https://github.com/yangyixxxx/gptnb-image) #6

> 在 Jupyter Notebook 环境中用自然语言控制图像生成与编辑的插件。由 **yangyixxxx** 开发，基于 GPT-4 和 Stable Diffusion。

<p><img src="https://opengraph.githubassets.com/1/yangyixxxx/gptnb-image" alt="gptnb-image" style="max-width:100%;border-radius:8px;"></p>

数据科学家常需在 Notebook 中快速生成配图或调整图像，传统方式需手动调用 SD API 或编写繁杂代码。**gptnb-image** 提供了一个 `%sd` 魔法命令，用户输入“生成一只坐在沙发上的猫，赛博朋克风格”即可自动完成。

插件解析自然语言指令，自动生成提示词（prompt），调用本地或远程 Stable Diffusion 模型，并将结果直接渲染到 Notebook 单元格输出。支持图像修复（inpainting）和风格迁移。

项目仅 8 个 Stars，但功能新颖。代码基于 **Python** 和 `diffusers` 库，适配 Jupyter Lab 4.0。开发者提供了详细的安装指南和 Demo 演示 Notebook。

相关链接：
- 项目地址：[yangyixxxx/gptnb-image](https://github.com/yangyixxxx/gptnb-image)
