## [YellowKey](https://github.com/Nightmare-Eclipse/YellowKey) #1

> 一款专注于轻量级加密存储的开源框架，由 **Nightmare-Eclipse** 团队维护。支持多平台密钥管理与对称/非对称加密混合方案，目前获得 **253** 个 Stars 和 **51** 个 Forks。

**YellowKey** 的设计目标是为开发者提供一套零依赖的密钥存储 API，避免在客户端应用中硬编码敏感凭据。项目采用模块化架构，底层使用 `libsodium` 加密库实现核心算法。

<p><img src="https://opengraph.githubassets.com/1/Nightmare-Eclipse/YellowKey" alt="YellowKey" style="max-width:100%;border-radius:8px;"></p>

核心亮点在于“即插即用”的集成体验：开发者只需提供主密码和存储路径，框架会自动生成密钥派生链，并加密写入本地文件。相比系统级 Keychain，**YellowKey** 更适用于跨平台桌面应用或嵌入式场景。

社区反馈显示，该项目在 **53** 个 Fork 中已有多个第三方分支用于游戏 Mod 的许可验证。团队定期发布安全更新，最近的 2.0 版本引入了基于 `Argon2` 的密钥拉伸，提升暴力破解难度。

相关链接：
- 项目地址：[Nightmare-Eclipse/YellowKey](https://github.com/Nightmare-Eclipse/YellowKey)

## [Umbrella-HWID](https://github.com/manojmidhul92-art/Umbrella-HWID) #2

> 一款由 **manojmidhul92-art** 开发的硬件标识符修改工具，使用 **C++** 编写，主要针对游戏反作弊系统。项目在 **203** 个 Stars 和仅 **2** 个 Forks 下获得较高关注度。

**Umbrella-HWID** 的核心原理是拦截操作系统对硬件信息的读取请求，通过驱动级 hook 动态篡改返回值，从而绕过基于硬盘序列号、网卡 MAC 地址等硬件的封禁机制。技术方案涉及 Windows 内核编程与 **WMI** 接口劫持。

<p><img src="https://opengraph.githubassets.com/1/manojmidhul92-art/Umbrella-HWID" alt="Umbrella-HWID" style="max-width:100%;border-radius:8px;"></p>

尽管 Forks 数量仅为 **2**，但该项目在反作弊社区中引发了密集讨论。开发者警告该工具仅用于教育研究，不建议用于破坏游戏公平性。从代码结构看，其支持对 **5** 大类硬件标识符的伪装，包括主板序列号与 TPM 芯片信息。

需要指出的是，使用此类工具可能违反多数游戏用户协议，甚至触及法律法规。项目目前未提供任何商业许可声明，仅以“研究目的”进行分发。

相关链接：
- 项目地址：[manojmidhul92-art/Umbrella-HWID](https://github.com/manojmidhul92-art/Umbrella-HWID)

## [GreenPlasma](https://github.com/Nightmare-Eclipse/GreenPlasma) #3

> 来自同一开发者 **Nightmare-Eclipse** 的又一安全工具，定位为系统级内存保护与反调试框架。获 **146** 个 Stars 和 **25** 个 Forks，与 YellowKey 形成互补生态。

**GreenPlasma** 专注于运行时保护，通过注入动态链接库监视常见的调试器特征（如 `Peb.BeingDebugged` 标志位），并在检测到调试行为时触发进程自毁或跳转至伪造执行路径。其核心引擎使用 **C++** 编写，并公开了部分 `VMP` 虚拟化指令的模拟实现。

<p><img src="https://opengraph.githubassets.com/1/Nightmare-Eclipse/GreenPlasma" alt="GreenPlasma" style="max-width:100%;border-radius:8px;"></p>

与同类项目相比，**GreenPlasma** 引入了“混淆定时器”机制，定期随机化内存校验间隔，增加动态分析难度。文档中提供了对 **x86** / **x64** 双架构的支持说明，并附有多个示例工程。

该项目在逆向工程领域有一定知名度，许多开发者将其作为学习反调试技术的参考源码。不过由于涉及对抗性技术，其更新频率低于 YellowKey，最后提交时间距今约 **4** 个月。

相关链接：
- 项目地址：[Nightmare-Eclipse/GreenPlasma](https://github.com/Nightmare-Eclipse/GreenPlasma)

## [skills](https://github.com/agentara/skills) #4

> **agentara** 团队开源的 AI 代理技能库，以静态 **HTML** 页面形式呈现。当前获 **141** 个 Stars 和 **3** 个 Forks，用于规范 AI 代理的能力描述与接口定义。

**skills** 项目旨在解决大语言模型代理（Agent）的能力发现难题。它定义了一套结构化技能描述模板，涵盖输入输出 schema、调用示例与前置条件，并以网页形式供其他系统解析引用。核心文件是 `skills/index.html`，内含 **20** 余种预定义技能。

<p><img src="https://opengraph.githubassets.com/1/agentara/skills" alt="skills" style="max-width:100%;border-radius:8px;"></p>

相比传统的 JSON Schema 格式，**skills** 选择纯 HTML 展示，降低了非技术人员理解门槛。Forks 数量虽少，但已被多个 Agent 框架（如 `Ara`）列为默认技能注册表来源。社区讨论焦点在于技能描述的版本化管理与动态更新机制。

该项目目前处于早期阶段，功能较为简单，但为 Agent 生态的互操作性提供了一个轻量级参考实现。开发者计划后续增加 YAML 导出与 REST API 接口。

相关链接：
- 项目地址：[agentara/skills](https://github.com/agentara/skills)

## [wangluoxiaoshuozhushou](https://github.com/showdownagain/wangluoxiaoshuozhushou) #5

> 一款面向中文网络小说作者的写作辅助工具，由 **showdownagain** 个人开发。获 **31** 个 Stars 和 **2** 个 Forks，提供素材管理与自动续写功能。

**wangluoxiaoshuozhushou** 集成本地词库、场景模板与角色关系图谱，作者可快速插入预设桥段或检查剧情逻辑。项目底层调用 **PaddleNLP** 进行简单文本生成，支持基于已有大纲的自动成段续写。前端采用 Vue 2 构建，后端为 Flask。

<p><img src="https://opengraph.githubassets.com/1/showdownagain/wangluoxiaoshuozhushou" alt="wangluoxiaoshuozhushou" style="max-width:100%;border-radius:8px;"></p>

核心亮点在于“冲突检测”模块：通过情感分析判定章节内的角色情绪曲线，避免出现不合逻辑的突然转变。该功能在社区反馈中受到好评，有作者表示可节省约 **30%** 的修改时间。

项目目前仍处于开发阶段，文档较为简略，但源码结构清晰，适合作为 NLP 入门与写作工具结合的学习项目。未来计划加入多平台云同步功能。

相关链接：

## [Galaxy-Bilibili-Downloader](https://github.com/JonathanNguyen1985/Galaxy-Bilibili-Downloader) #6

> 基于 **Python** 的 Bilibili 视频下载器，由 **JonathanNguyen1985** 维护。虽仅获 **8** 个 Stars 和 **6** 个 Forks，但凭借简洁的 CLI 设计与高清画质支持，在小众用户群中稳定迭代。

**Galaxy-Bilibili-Downloader** 通过解析 Bilibili 官方 API 获取视频流地址，支持 **1080P** 及更高画质（需用户账户权限）。项目使用 `requests` 与 `ffmpeg` 实现视频与音频的合并，并提供 `--batch` 参数批量下载合集内容。

<p><img src="https://opengraph.githubassets.com/1/JonathanNguyen1985/Galaxy-Bilibili-Downloader" alt="Galaxy-Bilibili-Downloader" style="max-width:100%;border-radius:8px;"></p>

其特色在于“弹幕嵌入”功能：可选择将 XML 弹幕文件转换为 ASS 字幕并与视频封装，保留观看体验。该项目代码量约 **800** 行，注释完整，适合作为 Python 网络爬虫与媒体处理的入门案例。

由于 Bilibili 频繁更新接口验证策略，该项目每 **2-3** 个月需要适配一次。开发者已在 Readme 中标注“仅供个人学习使用”，并内置了速率限制以避免触发风控。

相关链接：
- 项目地址：[JonathanNguyen1985/Galaxy-Bilibili-Downloader](https://github.com/JonathanNguyen1985/Galaxy-Bilibili-Downloader)
