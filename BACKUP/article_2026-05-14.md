## [invisible_playwright](https://github.com/feder-cr/invisible_playwright) #1

> **invisible_playwright** 是一个用于绕过网站机器人检测的 Python 库。它通过修改 Playwright 的自动特征，让无头浏览器行为看起来更像真实用户，从而避免被 Cloudflare 等反爬系统拦截。目前获得 86 Stars，13 Forks。

随着各大网站反爬升级，基于 Playwright 的自动化脚本常被快速识别并拦截。项目团队发现核心问题在于 WebDriver 特征、浏览器指纹等静态标记过于明显，因此开发了这套隐身增强方案。

<p><img src="https://opengraph.githubassets.com/1/feder-cr/invisible_playwright" alt="invisible_playwright" style="max-width:100%;border-radius:8px;"></p>

**invisible_playwright** 核心攻防点包括：移除 `navigator.webdriver` 属性、伪装 `plugins` 和 `languages` 列表、注入虚假的 WebGL 指纹。它还内置了随机 User-Agent 轮换和延迟操作模拟，让行为模式更接近人类。

该项目目前处于早期阶段，但社区反馈积极。对比同类方案（如 `puppeteer-extra` 的 Stealth 插件），**invisible_playwright** 更专注于 Python 生态，且 API 设计轻量，只需一行初始化代码即可启用隐身注入。

适用场景包括：需要频繁抓取受 Cloudflare 保护数据的价格监控、舆情分析、学术爬虫等。开发者需注意：本项目仅用于合法测试，滥用可能违反目标网站服务条款。

相关链接：
- 项目地址：[feder-cr/invisible_playwright](https://github.com/feder-cr/invisible_playwright)

## [polymarket-ai-trading](https://github.com/thinkpixelIab/polymarket-ai-trading) #2

> **polymarket-ai-trading** 是一个将 AI 模型接入 Polymarket 预测市场的自动化交易框架。它利用大语言模型解析新闻与事件信息，自动生成交易信号并在链上执行，项目当前获得 78 Stars。

Polymarket 作为去中心化预测市场，交易决策高度依赖信息解读速度。传统手动交易难以跟上实时事件流，而 AI 模型恰好擅长从非结构化数据中提取关键信号。

<p><img src="https://opengraph.githubassets.com/1/thinkpixelIab/polymarket-ai-trading" alt="polymarket-ai-trading" style="max-width:100%;border-radius:8px;"></p>

项目架构分为三层：**数据源层**（订阅新闻 RSS、Twitter 流和链上数据）、**分析层**（调用 OpenAI GPT-4、Claude 或本地模型进行事件影响评估）、**执行层**（通过 **Polymarket** API 自动下单）。用户可自定义评估提示词和风险参数。

目前项目无公开 Forks，但 Issues 区已有多次针对 API 限速和 gas 优化讨论。与同类项目（如 `PolymarketBot`）相比，**polymarket-ai-trading** 更强调模型插件的可替换性，支持从开源 LLM 到商业 API 的灵活切换。

需要注意的是，预测市场交易存在高风险，作者在 README 中明确提示项目仅供学习研究，不构成投资建议。运行前需准备以太坊钱包和相应 API Key。

相关链接：
- 项目地址：[thinkpixelIab/polymarket-ai-trading](https://github.com/thinkpixelIab/polymarket-ai-trading)

## [blooket-hacks](https://github.com/PrimeKeeper58/blooket-hacks) #3

> **blooket-hacks** 是一套针对教育游戏平台 Blooket 的浏览器端脚本合集，提供自动答题、无限代币、解锁稀有角色等功能。项目已获得 47 Stars，目前 Forks 为零，表明其仍在早期传播阶段。

Blooket 广泛应用于课堂互动，但部分学生寻求“捷径”获取游戏内货币或额外功能。项目作者收集并整理了多个开源脚本，覆盖 Blooket 的主流游戏模式（如 Gold Quest、Tower Defense）。

<p><img src="https://opengraph.githubassets.com/1/PrimeKeeper58/blooket-hacks" alt="blooket-hacks" style="max-width:100%;border-radius:8px;"></p>

这些脚本主要基于 **JavaScript**，通过注入 `fetch` 拦截修改请求响应，或在客户端覆盖游戏逻辑变量。例如 `auto-answer.js` 能自动解析题目并选择正确选项，`max-gold.js` 将金币数值固定为最大。

由于 Blooket 持续更新反作弊机制，这些脚本的有效性可能随时间衰减。项目没有提供持续维护承诺，使用者需自行承担账号封禁风险。教育工作者也应关注此类工具的传播，以便调整课堂监控策略。

相关链接：

## [AirTranslate](https://github.com/himomohi/AirTranslate) #4

> **AirTranslate** 是一款专为 macOS 设计的全局即时翻译工具，支持快捷键呼出翻译窗口、多翻译引擎切换，以及系统级取词翻译。项目基于 Swift 开发，stars 22，Forks 3。

macOS 原生翻译功能在跨语言工作流中有些局限，例如无法自定义引擎、缺少历史记录。**AirTranslate** 聚焦于“轻量 + 快速”，让用户在任何应用中选中文本后一键翻译。

<p><img src="https://opengraph.githubassets.com/1/himomohi/AirTranslate" alt="AirTranslate" style="max-width:100%;border-radius:8px;"></p>

技术实现上，**AirTranslate** 以菜单栏常驻应用形式运行，通过 `NSPasteboard` 监听剪贴板变化，或使用全局快捷键（默认 **Option+Space**）触发翻译弹窗。目前集成了 **DeepL**、**Google Translate** 和 **Apple 翻译**，用户可在偏好设置中选择默认引擎。

与同类工具（如 `Bob`、`Easydict`）相比，**AirTranslate** 目前功能较基础，但它原生支持 **macOS 14+** 的 `SwiftUI` 架构，界面响应流畅且内存占用低（约 < 30 MB）。开发者计划后续加入 OCR 识别和语音翻译。

该工具适合经常需要浏览外语文档、写邮件或阅读技术资料的 Mac 用户。当前版本暂不支持 Windows/Linux。

相关链接：

## [online-gencad-viewer](https://github.com/easyeda/online-gencad-viewer) #5

> **online-gencad-viewer** 是 EasyEDA 团队开源的浏览器端 Gerber、Excellon 文件查看器。它无需安装专业 EDA 软件，即可在线预览 PCB 设计文件，项目获 20 Stars。

传统 PCB 制造流程中，设计文件（Gerber RS-274X、钻孔文件）需要特定工具（如 **Altium**、**KiCad**）打开，协作时发送视图非常不便。**EasyEDA** 作为在线 EDA 平台，将核心渲染能力剥离为独立库开源。

<p><img src="https://opengraph.githubassets.com/1/easyeda/online-gencad-viewer" alt="online-gencad-viewer" style="max-width:100%;border-radius:8px;"></p>

项目基于 **TypeScript**，使用 Canvas 实现 2D 图形渲染，支持图层开关、测量、缩放和平移。它采用流式解析，可处理大体积文件（实测 50 MB 单文件无卡顿）。API 设计面向 Web 组件化，开发者可方便地嵌入到自己的网页或工具中。

与 **GerberViewer**（KiCad 配套工具）和 **Tracespace** 等开源方案对比，本项目的最大优势是零依赖——所有渲染逻辑自包含，无需安装任何 WASM 或后端服务。EasyEDA 团队表示将持续更新以支持最新的 Gerber 格式规范。

适用场景包括：PCB 工厂快速预览、团队协作审核、在线教学演示。开发者可通过 npm 包直接引用。

相关链接：
- 项目地址：[easyeda/online-gencad-viewer](https://github.com/easyeda/online-gencad-viewer)

## [ODE](https://github.com/JoeYing1019/ODE) #6

> **ODE** 是一个用 Python 编写的常微分方程数值求解库，专注于简单易用的 API 和基础方法覆盖。当前 Stars 16，Forks 1，适合教学和小型科学计算场景。

无论是物理模拟还是生物模型，ODE 求解都是常见需求。主流库如 `scipy.integrate` 功能强大但学习曲线陡峭，而 **ODE** 的目标是让初学者在几行代码内跑通自定义方程。

<p><img src="https://opengraph.githubassets.com/1/JoeYing1019/ODE" alt="ODE" style="max-width:100%;border-radius:8px;"></p>

项目实现了 **Euler**、**Runge-Kutta 4**、**Adams-Bashforth** 三种经典方法，支持单步和多步输出。用户只需定义导数函数和初始条件，传入时间数组即可获得数值解。代码完全纯 Python，无 **NumPy** 强制依赖（但推荐安装以提高性能）。

与 `odeint`、`solve_ivp` 相比，**ODE** 缺少自适应步长和事件检测功能，但胜在代码精简（核心模块 < 300 行），非常适合学生阅读源码理解算法原理。作者在 README 中提供了洛伦兹系统、SIR 模型等示例。

当前版本为 0.1.0，计划加入 **Dormand-Prince** 方法和刚性方程求解器。对于需要快速验证 ODE 想法的用户，这是一个干净的起点。

相关链接：
- 项目地址：[JoeYing1019/ODE](https://github.com/JoeYing1019/ODE)
