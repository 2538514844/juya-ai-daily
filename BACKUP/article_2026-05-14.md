## [invisible_playwright](https://github.com/feder-cr/invisible_playwright) #1

> 一个轻量级 Python 库，通过注入反检测补丁让 **Playwright** 自动化脚本绕过 Cloudflare、DataDome 等反爬系统，使浏览器指纹看起来更像真实用户。

<p><img src="https://opengraph.githubassets.com/1/feder-cr/invisible_playwright" alt="invisible_playwright" style="max-width:100%;border-radius:8px;"></p>

**网页自动化** 开发者常遇到被目标网站识别为机器人的难题，尤其是使用了 Cloudflare 或 DataDome 等高级反爬服务的站点。`invisible_playwright` 旨在通过底层 Hook 技术，在 Playwright 启动前修改浏览器环境变量，动态替换 `navigator.webdriver`、`navigator.plugins` 等关键属性，让自动化实例伪装成普通 Chrome 浏览器。

该项目采用 **Python** 编写，核心逻辑是生成一个补丁脚本并在页面加载前执行，覆盖上百个浏览器指纹检测点。据开发者测试，该方案能有效绕过主流 WAF 的机器人检测，且对性能影响极小。目前仓库已有 86 个 Star，13 个 Fork，社区反馈积极。

与同类项目 `puppeteer-extra-plugin-stealth` 相比，`invisible_playwright` 更专注于 Playwright 生态，配置更简洁——只需在启动浏览器时增加一行 `import invisible_playwright` 并调用 `patch_playwright()` 即可。适用于爬虫、自动化测试、数据采集等场景。

**相关链接**：
- 项目地址：[feder-cr/invisible_playwright](https://github.com/feder-cr/invisible_playwright)

## [Claude-Code-Design-AI](https://github.com/mikesheehan54/Claude-Code-Design-AI) #2

> 使用 **Claude API** 将前端 HTML/CSS/JS 代码自动解析并生成 **Figma** 可编辑设计稿，帮助开发者快速从代码还原设计原型。

<p><img src="https://opengraph.githubassets.com/1/mikesheehan54/Claude-Code-Design-AI" alt="Claude-Code-Design-AI" style="max-width:100%;border-radius:8px;"></p>

传统工作流中，从设计稿到代码是常见路径，但从已有代码反向生成设计稿却缺乏高效工具。`Claude-Code-Design-AI` 利用 **Claude 3.5 Sonnet** 模型的视觉与代码理解能力，分析前端文件中的 DOM 结构、CSS 样式和交互逻辑，生成对应的 Figma 节点层级。

工具以 **TypeScript** 编写，提供 CLI 和 API 两种使用方式。用户上传一个包含入口 HTML 的文件夹，工具会调用 Claude 模型输出 Figma JSON 文件，后者可直接导入 **Figma** 进行二次编辑。测试用例显示，对中等复杂度的登录页面，还原度超过 80%。

该项目刚发布不久，Star 数 84，Fork 数为 0，但已在开发者社区引发讨论。与 `html-to-design` 等工具相比，其核心优势在于利用大语言模型处理非标准布局和复杂样式，而非依赖固定规则。适合需要快速从旧项目提取设计规范、或做代码审查前端效果的场景。

**相关链接**：
- 项目地址：[mikesheehan54/Claude-Code-Design-AI](https://github.com/mikesheehan54/Claude-Code-Design-AI)

## [blooket-hacks](https://github.com/PrimeKeeper58/blooket-hacks) #3

> 针对教育游戏平台 **Blooket** 的一系列作弊脚本（JavaScript），支持自动答题、无限金币、修改游戏模式等，仅供学习研究。

<p><img src="https://opengraph.githubassets.com/1/PrimeKeeper58/blooket-hacks" alt="blooket-hacks" style="max-width:100%;border-radius:8px;"></p>

**Blooket** 是一款流行的课堂互动游戏平台，教师常用来做知识测验。但部分学生尝试寻找游戏漏洞以获取高分或游戏币。`blooket-hacks` 项目聚合了多个社区发现的 **Blooket** 前端接口注入技巧，提供一键运行的 Tampermonkey 脚本和书签工具。

项目以 **JavaScript** 实现，核心原理是拦截并篡改 Blooket 的 API 请求与客户端逻辑。例如，自动填充正确答案、跳过答题等待时间、直接修改玩家金币数量等。仓库中详细列出了每个 hack 对应的游戏模式，并附带风险提示：滥用可能导致账号封禁。

当前 Star 数 47，Fork 数 0。虽然项目名包含“hacks”，但其技术本质是 Web API 逆向与前端调试，对学习浏览器开发者工具、Ajax 拦截有一定参考价值。需注意：用于真实课堂会有作弊风险，作者声明仅供研究。

## [AirTranslate](https://github.com/himomohi/AirTranslate) #4

> macOS 菜单栏一键翻译工具，选中任意文本后按下快捷键即可调用 Google、DeepL 或 ChatGPT 翻译，结果以小弹窗展示。

<p><img src="https://opengraph.githubassets.com/1/himomohi/AirTranslate" alt="AirTranslate" style="max-width:100%;border-radius:8px;"></p>

**macOS** 原生缺少类似 Windows 上的“划词翻译”体验，现有工具如 Bob 已经停更或转向订阅制。`AirTranslate` 用 **Swift** 重写了菜单栏翻译应用，采用纯原生架构，内存占用极低。它监听系统全局快捷键（默认 `Cmd+Shift+T`），识别当前鼠标选中的文本，调用用户配置的翻译引擎。

支持 **Google Translate**、**DeepL**、**OpenAI ChatGPT** 三种后端，用户可在偏好设置中选择。翻译结果显示在菜单栏下拉的悬浮卡片中，支持朗读、复制和收藏。项目内置隐私模式，翻译内容仅在本地处理，不上传其他服务器。

目前 Star 22，Fork 3。与同类产品相比，`AirTranslate` 完全开源、无广告，且代码结构清晰，适合作为 macOS 原生开发学习案例。缺点是目前仅支持 macOS 12+，未提供 iOS 版本。

**相关链接**：
- 项目地址：[himomohi/AirTranslate](https://github.com/himomohi/AirTranslate)

## [online-gencad-viewer](https://github.com/easyeda/online-gencad-viewer) #5

> **立创EDA** 开源的在线 **Gerber**、**NC-Drill** 文件查看器，基于浏览器 WebGL 渲染，无需安装任何软件即可查看 PCB 制造文件。

<p><img src="https://opengraph.githubassets.com/1/easyeda/online-gencad-viewer" alt="online-gencad-viewer" style="max-width:100%;border-radius:8px;"></p>

PCB 设计完成后，工程师常需检查 Gerber 文件是否正确。传统的离线软件如 **CAM350** 或 **Altium** Viewer 体积大且收费。`online-gencad-viewer` 由 **EasyEDA**（立创 EDA）团队开发，使用 **TypeScript** 和 **WebGL** 在浏览器中直接解析并渲染 Gerber RS-274X 格式和 NC-Drill 文件。

工具支持分层显示、缩放、测量、DRC 检查等基础功能。用户只需将 `.gbr`、`.drl` 文件拖入浏览器窗口，即可秒级加载并预览 PCB 的铜皮、钻孔、阻焊等各层。项目完全开源，可自行部署或嵌入到其他 Web 应用中。

当前 Star 20，Fork 1。该查看器主要面向 **立创 EDA** 用户，但也兼容标准 Gerber 格式。与已有开源方案 `gerber-viewer` 相比，该工具在渲染性能和交互流畅度上更优。适合 PCB 打样前的快速检查，以及教育场景中的教学演示。

**相关链接**：
- 项目地址：[easyeda/online-gencad-viewer](https://github.com/easyeda/online-gencad-viewer)

## [ODE](https://github.com/JoeYing1019/ODE) #6

> 一个基于 **PyTorch** 的物理信息神经网络 (PINN) 框架，专门用于求解常微分方程 (ODE)，提供简洁的接口和可视化训练过程。

<p><img src="https://opengraph.githubassets.com/1/JoeYing1019/ODE" alt="ODE" style="max-width:100%;border-radius:8px;"></p>

传统数值方法（如 Runge-Kutta）求解 ODE 对于高维或复杂边界条件时效率下降，**PINN**（Physics-Informed Neural Networks）提供了一种数据驱动的替代方案。`ODE` 项目封装了从问题定义、网络构建到损失计算和可视化的完整流程，用户只需定义 ODE 方程和初始条件即可训练神经网络逼近解。

使用 **Python** 实现，基于 `PyTorch` 的自动微分机制构建物理损失项。项目内置了多种激活函数（Tanh、Swish、ReLU）和优化器（Adam、L-BFGS），并支持动态调整学习率。示例代码中包含了 `y' = -2xy`、`y'' + y = 0` 等经典方程的求解演示。

Star 16，Fork 1。相比成熟的库如 `deepxde`，`ODE` 更加轻量，专注于 ODE 场景而非偏微分方程，适合入门学习和快速原型验证。项目还提供了训练过程中的损失曲线和预测 vs 真值对比图，方便调试。

**相关链接**：
- 项目地址：[JoeYing1019/ODE](https://github.com/JoeYing1019/ODE)
