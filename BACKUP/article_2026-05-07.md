## [BetterNitroDiscord](https://github.com/WeritoP/BetterNitroDiscord) #1

> 一个模拟 **Discord Nitro** 功能的客户端增强脚本，让免费用户也能使用自定义表情、贴纸、高质量上传等付费特性。社区关注度较高，已获 **463 Stars** 和 **91 Forks**。

许多 Discord 用户希望享受 Nitro 带来的便利，但月费门槛限制了部分人群。**BetterNitroDiscord** 通过注入自定义样式和脚本，绕过客户端限制，模拟了 Nitro 的核心体验。

<p><img src="https://opengraph.githubassets.com/1/WeritoP/BetterNitroDiscord" alt="BetterNitroDiscord" style="max-width:100%;border-radius:8px;"></p>

该项目主要利用 **BetterDiscord** 框架进行插件开发，核心逻辑包括替换表情资源 URL、修改文件上传限制以及解锁贴纸库。用户只需将脚本放入 BetterDiscord 插件目录即可生效。

据社区反馈，该工具在 **468 Stars** 的加持下已成为同类项目中的热门选择。不过需要留意，模拟付费功能可能违反 Discord 服务条款，使用时需自行承担风险。

对于频繁使用自定义表情和高质量媒体分享的用户来说，**BetterNitroDiscord** 提供了一个性价比极高的临时方案。它与同类工具的区别在于持续更新维护，兼容最新版 Discord 客户端。

**相关链接**
- 项目地址：[WeritoP/BetterNitroDiscord](https://github.com/WeritoP/BetterNitroDiscord)

## [tokenspeed](https://github.com/lightseekorg/tokenspeed) #2

> 一个 **Python** 编写的命令行工具，用于测量大语言模型推理时的 **Token 生成速度**，支持 `vLLM`、`llama.cpp`、`TGI` 等多种后端。轻量、可重复，适合开发者横向对比模型性能。

大模型部署时，**每秒生成令牌数** 是衡量推理效率的核心指标。**tokenspeed** 由 **LightSeek** 组织开发，旨在提供标准化的基准测试流程，消除不同框架间的测量偏差。

<p><img src="https://opengraph.githubassets.com/1/lightseekorg/tokenspeed" alt="tokenspeed" style="max-width:100%;border-radius:8px;"></p>

工具采用 **异步请求** 模式，支持自定义 prompt 长度、并发数和输出长度。运行后直接输出平均 TPS（Tokens Per Second）以及首个令牌延迟，数据以表格形式呈现，便于对比。

目前 **tokenspeed** 在 GitHub 上获得 **240 Stars**，社区关注度在稳步上升。相比 **lm-eval** 等大型评测框架，它更聚焦于延迟和吞吐量，配置简单，适合快速定位推理瓶颈。

开发者可以利用它优化模型量化策略或选择更合适的推理引擎。对于生产环境的 **RAG** 或 **聊天应用**，该工具的测试结果能直接指导部署决策。

**相关链接**
- 项目地址：[lightseekorg/tokenspeed](https://github.com/lightseekorg/tokenspeed)

## [XHS_ALL_IN_ONE](https://github.com/cv-cat/XHS_ALL_IN_ONE) #3

> 一款基于 **JavaScript** 的小红书全合一工具，集成了内容爬取、笔记批量发布、数据分析与账号管理功能，覆盖社媒运营的完整链条。

小红书已成为品牌和内容创作者的重要阵地，但官方 API 限制严格，手动运营效率低下。**XHS_ALL_IN_ONE** 通过模拟网页端请求，将采集、发布和分析整合到一个工具中。

<p><img src="https://opengraph.githubassets.com/1/cv-cat/XHS_ALL_IN_ONE" alt="XHS_ALL_IN_ONE" style="max-width:100%;border-radius:8px;"></p>

项目核心包括：关键词搜索抓取、笔记详情提取、自动登录发布、以及互动数据统计。全部通过 **Node.js** 运行，配置文件采用 JSON 格式，可定制定时任务。

该项目自发布以来已收获 **131 Stars** 和 **74 Forks**，说明运营社群对此类工具需求旺盛。不过需要留意，批量操作可能触发平台反爬机制，建议合理控制频率。

对于中小型团队或独立博主，**XHS_ALL_IN_ONE** 提供了一个低成本的运营自动化入口。与同类工具相比，它覆盖面更全，但稳定性依赖于小红书前端接口的变动。

## [Zelda-Minish-Cap-PC](https://github.com/LuisAlexisSalazar/Zelda-Minish-Cap-PC) #4

> 经典 GBA 游戏《塞尔达传说 缩小帽》的 **PC 原生移植**版，使用 **C++** 完全重写，不依赖模拟器，支持现代操作系统。

任天堂并未官方发布该作的 PC 版本，但爱好者 **LuisAlexisSalazar** 通过反编译和代码重写，将原 GBA 逻辑迁移到桌面平台，保留原汁原味的像素画风与玩法。

<p><img src="https://opengraph.githubassets.com/1/LuisAlexisSalazar/Zelda-Minish-Cap-PC" alt="Zelda-Minish-Cap-PC" style="max-width:100%;border-radius:8px;"></p>

项目基于 **SDL2** 和 **OpenGL** 实现图形渲染与输入，支持窗口缩放、自定义按键和存档读档。目前虽然只有 **41 Stars**，但对于怀旧游戏社区意义重大，移植质量较高。

由于是爱好者项目，游戏逻辑可能存在细微差异，且需要用户自行提供原版 GBA ROM 中的资源文件。最终编译产物仅约 **10 MB**，运行流畅。

那些无法忍受模拟器延迟或希望获得原生体验的玩家，可以通过该项目直接在 **Windows / Linux** 上重温这款经典。它也为其它 GBA 游戏的 PC 移植提供了技术参考。

**相关链接**

## [AzuDL-GC2GD](https://github.com/TheGreatAzizi/AzuDL-GC2GD) #5

> 一个 **Python** 实用脚本，用于将 **Google Colab** 环境中的文件直接传输到 **Google Drive**，解决 Colab 会话结束后文件丢失的问题。

Google Colab 的临时实例在断开后会自动清除所有文件，用户需要手动挂载 Drive 并复制。**AzuDL-GC2GD** 将这一过程自动化，支持断点续传和目录结构保留。

<p><img src="https://opengraph.githubassets.com/1/TheGreatAzizi/AzuDL-GC2GD" alt="AzuDL-GC2GD" style="max-width:100%;border-radius:8px;"></p>

工具通过 **PyDrive2** 库操作 Google Drive API，核心功能只需一行命令 `python azudl.py --source /content/output --destination "MyFolder"`。同时支持限速和进度条显示。

该项目目前获得 **41 Stars**，虽然体量小，但解决的是数据科学家的高频痛点。对于训练模型、处理数据集后需要持久化的用户来说，这是一个省力的工具。

脚本还预留了扩展接口，允许用户自定义传输后的清理操作。与 Colab 内置的 `cp` 指令相比，**AzuDL-GC2GD** 提供了更友好的错误处理和重试机制。

**相关链接**
- 项目地址：[TheGreatAzizi/AzuDL-GC2GD](https://github.com/TheGreatAzizi/AzuDL-GC2GD)

## [tsl-scifi-earth](https://github.com/hexianWeb/tsl-scifi-earth) #6

> 基于 **Three.js** 构建的科幻风格 3D 地球可视化项目，带有动态粒子光环、发光轮廓和星空背景，适合作为数据大屏或游戏背景。

传统 Web 地球可视化往往偏重地理数据展示，而 **tsl-scifi-earth** 更强调视觉冲击力。作者 **hexianWeb** 利用 Shader 与后处理特效，打造了一个赛博朋克风格的地球模型。

<p><img src="https://opengraph.githubassets.com/1/hexianWeb/tsl-scifi-earth" alt="tsl-scifi-earth" style="max-width:100%;border-radius:8px;"></p>

项目使用 **JavaScript** 编写，核心代码不到 300 行，通过 `OrbitControls` 实现鼠标拖拽旋转。支持自定义颜色主题和粒子密度，可嵌入任何网页。

尽管 **Stars** 仅 16，但该项目的视觉效果在社交平台上获得不少分享。对于前端开发者或数字艺术爱好者来说，它提供了一个可直接运行的学习范例。

与功能复杂的 **Cesium** 或 **Mapbox** 不同，**tsl-scifi-earth** 专注于风格化渲染，适合非专业地图用途的演示。未来可能加入动画过渡和数据接入能力。

**相关链接**
- 项目地址：[hexianWeb/tsl-scifi-earth](https://github.com/hexianWeb/tsl-scifi-earth)
